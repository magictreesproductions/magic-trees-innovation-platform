## Key Principles

### 1. **Problem-First, Not Tech-First**
Judges don't care about your technology. They care about the problem you're solving and whether your solution actually solves it.

### 2. **Execution > Perfection**
A working prototype beats a theoretical masterpiece every time. Build real things.

### 3. **Learn, Then Iterate**
After each hackathon, capture what worked. Your win rate improves 2-3% per event if you actually learn.

### 4. **Serve, Don't Seek**
The hackathons you win are the ones where you're genuinely trying to help someone. Judges can smell inauthenticity.

### 5. **Momentum is Multiplicative**
Each win funds the next project. Each project teaches you something. Compound this over 12 months and watch what happens.
